# Signal Flow 浮动阅读导航设计

## 目标

在 Signal Flow 滚动区域右侧增加浮动阅读导航，方便用户快速回到最新消息、恢复上次阅读位置或跳到当前列表最早消息。

## 默认行为

- 页面打开后仍停留在最新消息，不自动跳转到历史阅读位置。
- TG/X 新消息到达时，保留现有阅读位置稳定逻辑，不打断阅读。
- 用户需要恢复历史阅读位置时主动点击按钮。

## 导航布局

- 采用 B2 方案：桌面端显示文字按钮，手机端缩为图标。
- 导航固定在 Signal Flow 滚动区域右侧中部，靠近滚动条。
- 不跟随消息卡片移动，不挤占列表宽度。
- 手机端按钮提供 `aria-label` 和浏览器 tooltip。

## 按钮

1. `↑ 最新`
   - 滚动到当前列表第一条消息。
   - 用户离开列表顶部后显示。
   - 有新消息时显示数量角标。

2. `↩ 上次阅读`
   - 复用本机 `localStorage` 保存的阅读锚点。
   - 找不到旧消息时保留当前位置，并显示提示。

3. `↓ 最早消息`
   - 滚动到当前筛选列表最后一条消息。
   - 当前列表为空时不执行滚动。

## 兼容

- 桌面端兼容 Signal Flow 内部滚动容器。
- 手机端兼容页面滚动。
- 切换标签、时间范围、作者筛选或搜索后，浮动导航只操作当前筛选结果。
- 顶部工具栏现有“返回上次阅读”按钮保留，作为冗余入口。

## 测试

- 浮动导航组件契约测试：三个按钮、桌面文字、手机图标、可访问标签和未读角标。
- Signal Flow 契约测试：浮动导航接收当前列表、未读数量和回调。
- 阅读位置纯函数测试继续通过。
- ESLint 和生产构建通过。
